# 터미널 1:1 채팅 사용법 (Windows)

간단한 터미널 채팅 프로그램이에요. 메시지는 **끝에서 끝까지 암호화**(E2E)
되어서 서버도 내용을 못 봐요. 아래 순서대로 **한 번만** 세팅하면 돼요.
이후엔 `node chat.js 방이름` 한 줄로 실행.

---

## 1단계. Node.js 설치 (최초 1회)

1. https://nodejs.org/ 접속
2. **LTS 버전** (왼쪽 녹색 버튼) 다운로드 — **22 이상** 필수
3. 다운받은 설치 파일 실행 → 전부 **Next** → **Install**
4. 설치 완료

**설치 확인**:
- 윈도우 검색창에 `PowerShell` 입력 → 실행
- 아래 명령어 복사해서 붙여넣고 Enter:
  ```powershell
  node -v
  ```
- `v22.x.x` 이상이 나오면 OK. 안 나오면 컴퓨터 재시작 후 다시 시도.

---

## 2단계. 채팅 파일 준비 (최초 1회)

1. **받은 zip 파일** (`chat-cli.zip`)을 **바탕화면**에 저장
2. 우클릭 → **압축 풀기** → 폴더 생김 (예: `chat-cli`)
3. PowerShell에서 그 폴더로 이동:
   ```powershell
   cd $HOME\Desktop\chat-cli
   ```

**끝입니다.** `npm install` 없어요 (Node 22+ 내장 WebSocket 사용).

---

## 3단계. 채팅하기

### 방법 A. 고정 방 (추천 — 매번 같은 명령만)

상대와 **방 이름**을 미리 정합니다 (예: `우리둘만-7k4x`).
둘이 **같은 명령**을 치면 자동 연결:

```powershell
cd $HOME\Desktop\chat-cli
node chat.js 우리둘만-7k4x
```

첫 실행 시 **이름**을 한 번 묻습니다. 이후 저장돼서 안 물어봐요.

> 방 이름은 **타인이 추측 어려운 조합** 권장 (숫자+영문 조합).
> 혹시 모를 충돌 시 **세이프티 코드**(연결 시 표시되는 8자)가 서로 다르면 바로 감지됩니다.

### 방법 B. 상대가 초대링크 보낸 경우

```powershell
cd $HOME\Desktop\chat-cli
node chat.js "wss://chat-cli-7woy.onrender.com#받은토큰"
```
※ 따옴표 `""` 꼭 붙여야 해요. `#` 때문에 없으면 오류.

### 연결 후

- 프롬프트 `[내이름] >` 뜨면 입력 → Enter로 전송
- 명령어는 `/help` 입력

### 종료

- **Ctrl + C** 누르기 또는 `/quit`

## 자주 겪는 문제

| 증상 | 해결 |
|------|------|
| `node : 용어가 인식되지 않습니다` | Node 설치 안 됐거나 재시작 안 한 상태. 컴퓨터 재부팅. |
| `Node.js 22 이상이 필요합니다` | nodejs.org에서 **최신 LTS** 다시 설치. |
| 첫 연결이 30~60초 걸림 | 정상. 서버가 자고 있다가 깨는 중. 기다리면 연결됨. |
| `room full` | 방에 이미 2명 있음. 다른 방 이름 쓰거나 한 명 나가길 기다려야 함. |
| 연결 끊김 / 멈춤 | Ctrl+C로 끄고 다시 실행. 같은 방 이름이면 재접속. |

## 유용한 명령어 (채팅 중에)

| 명령 | 설명 |
|------|------|
| `/help` | 도움말 |
| `/clear` | 화면 + 스크롤백 비우기 |
| `/name 새이름` | 내 이름 바꾸기 |
| `/color me` / `/color peer` | 색 바꾸기 (빨·주·노·초·파·남·보·흰·레인보우) |
| `/bell` | 상대 메시지 올 때 소리 on/off |
| `/quit` | 종료 |

---

## 참고

- **방 이름**: 한 번 정하면 계속 쓰세요. 굳이 바꿀 필요 없음.
- **히스토리 저장 안 됨**: 창 닫으면 내용 사라짐 (E2E 원칙상).
- **한 방에 2명만**: 제3자 접속 자동 차단.
- **네트워크**: HTTPS로 통신해서 대부분 방화벽 통과.
